Product Ordering Methods

How Compify decides the order in which products are presented to each judge.

━ Linear

Products are presented in the same fixed order to every judge. Simple and predictable — the classic approach for traditional competitions.

Pros

  • Simple to understand and set up
  • Easy to track which product is being evaluated
  • Consistent experience for all judges
  • Ideal for small competitions

Cons

  • Palate fatigue affects later products
  • Position bias: first and last products are rated differently
  • Not suitable for large-scale blind tastings

⟠ Randomized

Each judge receives products in a completely random order. Eliminates position bias by distributing palate fatigue effects evenly across all entries.

Pros

  • Eliminates position bias completely
  • Distributes palate fatigue effects evenly
  • Statistically fair for all entries
  • Works well for any competition size

Cons

  • Harder to coordinate logistics (stewards need per-judge order)
  • Requires more careful table management
  • Harder to troubleshoot if issues arise mid-session

◆ Algorithm-guided

Compify's proprietary algorithm optimizes product order per judge, balancing multiple factors: category diversity, palate reset needs, producer separation, and statistical fairness. The most advanced method.

Pros

  • Maximizes evaluation precision mathematically
  • Balances multiple fairness criteria simultaneously
  • Separates same-producer products automatically
  • Optimizes palate fatigue recovery
  • Best results for high-stakes competitions

Cons

  • Less transparent to judges (order logic is algorithmic)
  • Requires full product data to be entered in advance
  • Overkill for small, informal competitions
